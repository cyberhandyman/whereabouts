# 本地化(i18n)指南

本项目用 **Xcode 15+ String Catalog**(`Whereabouts/Localizable.xcstrings`)管理 UI 字符串。开发语言是 `zh-Hans`,英文为第二语言。新增任何用户可见文本前,请通读本文。

---

## 0. 速查 / TL;DR

- 字面量 → 用 catalog key,**别**留中文/英文裸字符串
- SwiftUI View 内:`Text("my.feature.title")` —— SwiftUI 自动走 `LocalizedStringKey`
- 非 view 代码:`String(localized: "my.feature.title")`
- 动态 String 当 key:`Text(LocalizedStringKey(stringVar))`
- 带变量:`Text("my.key \(arg)")`,catalog 里 key 是 `"my.key %@"` / `"%lld"`
- 多变量重排序:`%1$@` / `%2$@`(英文可改词序)
- 复数:用 catalog 的 plural variations
- 写完后 **build 一次**,Xcode 会自动把新 key 收集到 catalog

## 2. Key 命名规范

格式:`<feature>.<component>.<element>[.<variant>]` —— **dot.notation**、**全小写**、**语义化英文**。

| 段 | 例子 | 说明 |
| --- | --- | --- |
| `feature` | `app / input / search / filter / empty / detail / edit / refresh / dup / update / settings / meta / time / location / action / error / menu` | 顶层分组,按 view / 功能区 |
| `component` | `alert / button / textField / section / label / placeholder / message / chip / field / toggle / menu / photo / used / history / meta` | view 类型或子区域 |
| `element` | `title / description / tooltip / count / placeholder ...` | 具体元素 |
| `variant` | 可选 | plural / 状态 / 不同精度 |

**例子**:

- `app.name` — 顶级标题
- `edit.section.basic` — 编辑表单"基本"section
- `edit.field.model.placeholder` — 编辑表单"型号"字段占位符
- `detail.history.count` — 详情页历史计数(plural)
- `filter.chip.model %@` — 筛选 chip,带变量

### 跨 view 复用 / 共享 key 的命名空间

| 命名空间 | 用途 | 例子 |
| --- | --- | --- |
| `action.*` | 通用动作按钮 | `action.cancel`、`action.done`、`action.delete`、`action.edit` |
| `meta.label.*` | 物品元数据字段标签(型号/颜色/...) | `meta.label.model`、`meta.label.color` |
| `location.*` | 位置相关共享文案 | `location.unspecified` |
| `time.*` | 相对时间文案 | `time.justNow` |
| `error.*` | (预留)错误文案,Phase 5 iCloud / 照片相关 | — |
| `menu.*` | (预留)Phase 6 macOS 菜单栏 | — |

复用一个共享 key 比每个 view 起独立 key **更好** —— 翻译一次,多处生效,中英表里也少冗余。

## 3. 在代码里引用

### 3.1 SwiftUI View(最常见)

SwiftUI 的 Text / Label / Button / TextField / Section / .navigationTitle / .help 等 init 都接受 `LocalizedStringKey`,**字面量自动走 catalog**:

```swift
Text("detail.history.title")                              // 简单 key
Text("input.preview.willCreate \(count)")                 // 带变量(catalog key 是 "input.preview.willCreate %lld")
Text("dup.alert.message.withLocation \(name) \(old) \(new) \(path)")  // 多变量
TextField("edit.field.name", text: $item.name)
Button("action.cancel") { ... }
Label("detail.button.delete", systemImage: "trash")
.navigationTitle("edit.navTitle")
.help("detail.photo.tooltip")
```

### 3.2 字符串变量当 key(动态 key)

当 key 本身存在 `String` 变量里(比如循环里),用 `LocalizedStringKey` 显式包:

```swift
let chips: [(labelKey: String, value: String)] = [
    ("meta.label.model", item.model),
    ("meta.label.color", item.color),
]

ForEach(0..<chips.count, id: \.self) { i in
    Text(LocalizedStringKey(chips[i].labelKey))    // 显式包一层 → 走 catalog
    Text(chips[i].value)                            // value 是用户数据,verbatim
}
```

⚠️ **陷阱**:`Text(chips[i].labelKey)` 会走 `Text(_ content: some StringProtocol)` overload,verbatim 显示(不查 catalog)。一定要 wrap。

### 3.3 非 view 代码(普通 String)

用 `String(localized:)`:

```swift
let summary = String(localized: "update.summary.model \(value)")
let ack = String(localized: "detail.used.ack.kept")

// 在 Optional fallback 里
Text(log.location?.path ?? String(localized: "location.unspecified"))
```

### 3.4 多个本地化片段拼接

**不要**手写分隔符。用 `ListFormatter`,locale-aware(中文 `A、B`,英文 `A and B`):

```swift
let parts = [String(localized: "..."), String(localized: "...")]
let joined = ListFormatter.localizedString(byJoining: parts)
```

### 3.5 日期 / 数字

**永远不要**手写格式化字符串。用 `Date.FormatStyle` / `IntegerFormatStyle`:

```swift
date.formatted(.dateTime.year().month())     // zh: "2024年5月" / en: "May 2024"
count.formatted(.number)
```

`formatPurchaseDate` 在 `Models/Item.swift` 已示范用法。

### 3.6 仅展示原始字符串(用户数据)

用户输入(`item.name` / `item.notes` / location path 等)是数据,**不翻译**。显示时:

```swift
Text(item.name)                       // String overload,verbatim
Text(verbatim: "\(item.count)")       // 显式 verbatim,绕开 LocalizedStringKey "%lld" 自动收集
```

## 4. Catalog 里加 / 改 entry

`Localizable.xcstrings` 是 JSON。手编时遵守:

### 4.1 简单字符串

```json
"edit.field.name" : {
  "comment" : "Edit form: name field label.",
  "localizations" : {
    "en" :     { "stringUnit" : { "state" : "translated", "value" : "Name" } },
    "zh-Hans" : { "stringUnit" : { "state" : "translated", "value" : "名称" } }
  }
}
```

### 4.2 带变量(单)

key 字符串里包含 placeholder format specifier:

```json
"refresh.title %@" : {
  "localizations" : {
    "en" :     { "stringUnit" : { "state" : "translated", "value" : "Re-parsing \"%@\":" } },
    "zh-Hans" : { "stringUnit" : { "state" : "translated", "value" : "根据「%@」重新看了一下:" } }
  }
}
```

调用:`Text("refresh.title \(item.name)")`。

### 4.3 带变量(多 / 可重排序)

用 positional `%1$@`、`%2$@`、`%3$@`、`%4$@` —— 这样英文翻译可以重排词序:

```json
"dup.alert.message.noLocation" : {
  "localizations" : {
    "en" :     { "value" : "\"%1$@\" is already at %2$@.\nYou didn't say where it is now — want to add %3$@ to it, or create a new one?" },
    "zh-Hans" : { "value" : "已有「%1$@」在 %2$@。\n这句话没说新位置 —— 是想给它补充信息(%3$@),还是另起一条?" }
  }
}
```

调用:`Text("dup.alert.message.noLocation \(name) \(oldLoc) \(meta)")`。

### 4.4 Plural(单复数)

英文要 `one / other`,中文不分单复数只填 `other`:

```json
"detail.history.count" : {
  "localizations" : {
    "en" : {
      "variations" : {
        "plural" : {
          "one"   : { "stringUnit" : { "state" : "translated", "value" : "%lld time"  } },
          "other" : { "stringUnit" : { "state" : "translated", "value" : "%lld times" } }
        }
      }
    },
    "zh-Hans" : {
      "variations" : {
        "plural" : {
          "other" : { "stringUnit" : { "state" : "translated", "value" : "%lld 次" } }
        }
      }
    }
  }
}
```

调用:`Text("detail.history.count \(count)")`。

### 4.5 Xcode 自动同步

- build 时,Xcode 扫描代码里所有 `Text("...")` / `String(localized: "...")` 使用,把发现的 key 自动加入 catalog(`extractionState = "extracted_with_value"`)。
- 你手编添加的 entry(`extractionState` 不写)Xcode 不会覆盖。
- 代码里已删除但 catalog 里还在的 key 会被标 `stale` —— 看到了就清。

最佳实践:**写代码 → build → 在 Xcode UI 里打开 .xcstrings → 看新加的 key → 补翻译**。Xcode 的 catalog 编辑器比手编 JSON 友好得多。

## 5. 不本地化的部分(明确边界)

| 内容 | 原因 |
| --- | --- |
| `InputParser` 词典 / 分隔符 / 模式串 | 中文 NLP 数据,英文输入解析是另一套(Phase 7+ 用 LLM 兜底,见 §7) |
| 用户输入(`item.name` 等) | 数据本身,不翻译 |
| SF Symbol 名 (`"trash"` 等) | 系统图标 ID |
| `purchaseDatePrecision` 字段值 (`"year"/"month"/"day"`) | 内部 enum 字串,只是持久化值 |
| 注释 `///` / `// MARK:` | 写给开发者,中文留着 |
| `\u{FF0C}` / `\u{3001}` 等 unicode 转义 | 字符常量,不是 UI |

## 6. 运行时切换语言

- **Settings 入口**:菜单栏 `Whereabouts → Settings…`(⌘,);三选 `Follow System / Chinese (Simplified) / English`
- **数据安全**:切换只影响 *显示*,不动 SwiftData 任何字段
- **生效范围**:
  - `.environment(\.locale, ...)`(在 `WhereaboutsApp.body`):SwiftUI Text / LocalizedStringKey **立刻**切换
  - `AppleLanguages` UserDefaults(写入 `["en"]` / `["zh-Hans"]`):非 view 代码里 `String(localized:)` 用的 Bundle.main lookup —— **需 restart 才完全跟随**
- 因此 Settings 改语言后,显示"重启应用以完全生效"提示

## 7. InputParser 与 i18n 的关系

`InputParser` 当前是 **中文 NLP**:词典 / 规则全部为中文输入设计。英文输入会进入 parser 但抽不出元数据,整句落到 `item.name` 字段。

- 文件顶部有 `TODO: i18n - English parsing via LLM (planned post-Phase 7)` 注释
- 调用点(ContentView.commit / ItemDetailView.saveNewLocation 等)也加了同样 TODO
- 面向用户的 summary 字符串(`buildFieldUpdate` / `makeBuySummary`)走 `String(localized: "update.summary.*")`,可以本地化

未来扩 LLM 兜底时,在主入口前加 dispatch:

```swift
let locale = Locale.current.language.languageCode?.identifier
if locale == "zh" { /* current parser */ } else { /* LLM */ }
```

## 8. Accessibility(规范预留)

本次没做 VoiceOver 标签迁移,但**新增交互式 View 必须**:

```swift
Button { ... } label: { Image(systemName: "...") }
    .accessibilityLabel("my.feature.action.label")    // catalog key
```

且 `.accessibilityLabel` 接受 `LocalizedStringKey`,**直接传 catalog key 字符串**,文本走 catalog,不要写裸字符串。

---

## 9. 边界 / 不做的

- **不支持 RTL**(阿拉伯/希伯来等右到左)
- **不分地区**(`en-US` / `en-GB` 统一 `en`;`zh-Hans` / `zh-Hant` 暂只支持 `zh-Hans`)
- **不引入**第三方 i18n 库(Polyglot / Lingo 之类)
