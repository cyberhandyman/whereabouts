何处 Whereabouts  ·  使用指南

本指南同时适用于 Mac 和 iOS(iPhone / iPad)版。大部分功能通用,
只是操作方式不同——Mac 用鼠标右键 / 快捷键的地方,iOS 对应长按 / 滑动手势。
iOS 专属内容见「📱 iOS 版」一节;标注 (macOS) 的章节仅 Mac 版适用。

  📝  录入物品
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

在顶部输入框写一句话,回车提交。

支持的句式:
  · 充电宝 在 卧室抽屉
  · 护照 → 保险箱
  · 双立人菜刀是 2024 年在山姆买的
  · 书房桌上有 AITO 专项游戏采集底座
  · 床头柜上的钥匙

自动抽取:型号、颜色、购买日期、购买渠道、容量/规格。

一次记多条:换行、分号、逗号、顿号、「然后」「以及」都能切分。
  · 钥匙在玄关,雨伞在门口
  · 3 个 Magic Keyboard 键盘在书房;2 副耳机在卧室

数量量词会原样保留(不会被自动删掉)。

多件物品共用一个位置:
  写 "iPhone、AirPods、Apple Watch 在书房抽屉" 时,
  三件物品都会挂到"书房抽屉"——不只是第一件。
  同一次输入里没有写位置的物品会自动继承最近那条明确填了位置的记录。

注意:中文解析,英文输入整句进物品名。
英文模式:如果中文拆分没有命中,解析器会自动尝试英文句式,
  支持 "X in Y" / "X at Y" / "Y has X" 等格式。

位置自动完成:
  · 输入框下方显示最近用过的位置 chip,点击追加到输入框
  · 文本里提到已存在的房间名 → 上方再出现「<房间> 内」chip 行,
    列出该房间的子位置,点击可直接使用
  · 位置匹配不区分大小写、全/半角:"hifi" 能命中 "HiFi"

没有配置 AI 的情况下,输入框下方会出现紫色胶囊提示。
点击直接跳转到 偏好设置 → AI 完成配置。

  ✨  AI 理解
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

本地解析覆盖不到的奇怪命名、混写地点,可以让 AI 重新理解。

配置 AI(⌘, → AI tab):
  · Provider:Claude 或 火山引擎(两个可同时配置,随时切换)
  · 模型:Haiku 4.5 / Sonnet 4.6 / Sonnet 5 / Opus 4.7 / Opus 4.8(Claude)
           或填火山引擎模型名 / endpoint ID
  · API key:填入对应平台的密钥
  · Endpoint:默认官方地址,可改成中转站 / 代理 URL
  · 「测试连接」按钮:1 秒内验证 key 是否有效
  · 「查看图文配置教程(网页)」链接:不熟悉 Claude / 火山引擎账号注册、
    充值、拿 key 流程的话,点开跟着教程一步步操作,教程里还有费用
    估算和报错排查。中英双语,Mac 和 iOS 都有这个入口。

> ✅ v0.2.0:Claude 模型新增 Sonnet 5 和 Opus 4.8 两档;
> AI 配置区新增「查看图文配置教程(网页)」入口。

触发 AI 理解的三种方式:
  1. 录入时勾「使用 AI 理解」
     每条录入先走本地解析瞬间落库,之后 AI 在后台异步跑。
     该行右侧显示 ✨ 旋转图标 + 「正在用 AI 理解…」,
     完成后变成 ✅「AI 已理解 ✓」,4 秒后自动消失。

  2. 详情页 → ✨ 用 AI 理解 按钮
     单条立即运行,不阻塞 UI。

  3. 多选 → 右键菜单 → ✨ 用 AI 理解
     批量后台处理,每条物品行内显示进度,可以继续其它操作。

AI 做了什么:
  · 语义级重新解析所有文字字段(名称、位置、型号、颜色等)
  · 把你录入时写的整段原文作为最高优先级参考——
    多条混写、含糊地点时 AI 能更准确地还原你的意思
  · 多件物品同时输入时,AI 只修改当前这件,不会影响其他条目的名称
  · 数量量词(「3 个」「2 副」等)会原样保留,不会被 AI 删掉
  · 必须从你已有的标签里挑一个挂上;都不合适才挂「其他」——
    AI 永远不会自创新标签
  · AI 的标签替换当前所有 tag(不叠加)
  · AI 返回 null 的字段保持原值,不抹掉已有数据

AI 位置容错:
  AI 能看到你本机所有现有位置,遇到拼写偏差时会自动复用最相近的已有位置,
  而不是建一个新的。容错范围涵盖:1-2 字符差异、同音字、繁简转换、
  大小写差异、全角/半角差异。
  例:输 "塑料袋抽屉",已有 "塑料抽屉" → AI 直接复用"塑料抽屉"。

还原 AI 修改:
  详情页历史时间线里,AI 修改 name 的那一行右侧有 ↩ 还原 按钮。
  点一下可改回 AI 修改之前的名字。

AI 用量统计(偏好设置 → AI):
  顶部用量 section 分三列展示:今天 / 本周 / 本月。
  每列显示:调用次数 / input token / output token / Claude 估算 USD。
  月初自动归零;也可手动点「重置」清空。
  口径说明:token 数直接取自 API 响应的 usage 字段,
  与 Anthropic / 火山引擎账单口径一致。

火山引擎用户可在 AI tab 的火山引擎 section 里填「输入 ¥/百万 token」
和「输出 ¥/百万 token」两个可选价格字段(单价可在火山方舟控制台各 model
详情页查到)。填写后用量统计区会多一行 ¥ 估算,跟 Claude 的 $ 估算并列
显示;留空则只显示 calls / tokens。

隐私说明:
  仅在你主动点击 ✨ 时才发起请求。
  不发送照片和历史 log,只发当前物品的文字字段。

点击列表中一项打开详情(右侧 inspector)。

右上角按钮:
  ✨ 用 AI 理解  · 调 AI 重新解析字段(需配置 API key)
  ✏️ 编辑        · 打开完整编辑表单

「用过这个吗?」四个选项:
  · ta 还在原位  → 确认仍在原位,写一条历史
  · 位置变了…   → 行内修改位置(带自动完成 chip 提示)
                   点击 chip 会直接替换输入框里的内容(不是追加)
  · 放回去了     → 标记已取用后放回
  · 不管了       → 暂时找不到,清空位置但保留历史

借出物品:
  详情页底部有紫色「借给…」按钮。
  点击后填写借出人姓名,确认后列表行显示紫色「借给:XX」标记,
  详情页顶部出现橙色「已借给 XX」徽章。
  归还时点徽章或右键菜单 → 标记已归还。
  借出和归还会作为时间线事件记录,显示「借出去 · 借给 XX」紫色行
  和「已归还 · ← XX」橙色行。

搜索框也能直接搜借出人姓名,筛选面板里的「借出」facet 可以一键看所有
借出去或在家的物品,详见「搜索与筛选」章节。

关联物品:
  · 该物品所在的关联组里,其余物品以蓝色超链接列出
  · 点击任意一件,inspector 立即切到对方的详情

底部时间线:位置变化 + 字段编辑历史合并显示
  · 蓝色  = 本地解析
  · 绿色  = 手工编辑
  · 紫色  = AI 修改 / 借出事件
  · 橙色  = 用户还原 / 归还事件
  · AI 修改名称的那行右侧有 ↩ 还原 按钮

底部:删除(进回收站,可还原)

  🔍  搜索与筛选
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

搜索框和录入框始终钉在顶部,展开筛选下拉时只压缩物品列表,顶部区域不移动。

搜索框旁边的 chevron 按钮控制筛选行的折叠与展开。
筛选下拉最高 280pt,位置很多时内部自动滚动。

搜索框模糊匹配:名字 / 位置 / 型号 / 颜色 / 渠道 / 借出人 / 标签名。

六行 facet 筛选:
  房间  · 只列根级位置(书房、客厅…)
          计数 = 该房间整棵子树下的物品数
          点击 → 显示该房间所有物品(含所有子位置)
  位置  · 只列非根级位置及没有房间的孤儿位置
          以完整路径显示(如「书房 > 收纳抽屉」)
          点击 → 显示精确在该路径上的物品
  渠道  · 京东 / 山姆 / 闲鱼 等
  年份  · 按购买年份
  品牌  · 从名字自动推断(不入库)
  借出  · 仅当至少一件物品被借出时才显示,有两个 chip:
          「借出去 N」— 显示当前所有不在家的物品
          「在家 M」  — 排掉借出去的,只看在家的物品

房间和位置可以同时选中:房间筛整棵子树,位置精确定位到某个节点。
点 chip 切换筛选,再点取消。「清除全部」一键重置。

已选中的筛选 chip 即使折叠筛选行也保留显示,
让你随时知道当前生效的过滤条件。

数据自动维护:
  每次启动时 app 会自动检查并清理两类历史遗留数据——
  ① 名称里含路径分隔符的脏位置(如早期 AI 偶尔产生的"书房 > 抽屉"整段当根)
  ② 大小写不同导致的同名房间重复(如两个"书房")
  有清理时底部会弹 toast 告知影响数量。
  清理完全自动,幂等无开销,干净数据不触发任何提示。

  📱  iOS 版(iPhone / iPad)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

需要 iOS 17 及以上系统。

三个标签页:
  · 物品    列表 + 搜索 + 房间筛选 chips + 统计瓷砖
  · 记一条  自然语言录入,解析规则 / 重复检测 / 消歧提示
            与 Mac 版完全相同
  · 设置    通用 / 录入行为 / 通知 / AI / 数据 / 关于
            各分类的开关和字段跟 Mac 版对应功能一致(见前面各节),
            只是按手机屏幕重新分了组

列表手势:
  · 左滑    置顶
  · 右滑    删除 或 编辑
  · 长按    弹出菜单:置顶 / 编辑 / 借给… / 标记已归还 /
            ✨ 用 AI 理解 / 删除

详情页:
  · 顶部位置面包屑,一路显示到根位置
  · 「用过吗?」五个快捷动作按钮:四个位置确认
    (还在原位 / 放回原位了 / 位置变了 / 不知道在哪)
    加一个整宽紫色「借给…」;归还在顶部橙色
    「已借给 XX」徽章里点「归还」(或长按列表行选归还)
  · 位置变化和字段编辑历史合并成一条时间线(与 Mac 版一致)
  · 关联物品以蓝色链接列出,点击直接跳转到对方详情
  · 照片支持全屏查看,双指缩放

数据存储:
  iOS 版和 Mac 版数据各自独立保存在本机,不会自动同步。
  要在两台设备间搬运数据,用 设置 → 数据 里的 JSON 导出 / 导入——
  先在一端导出,把文件传到另一端,再导入即可(见下一节)。


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  💾  数据导出与导入
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

导出:主窗口右上 ⬆ 按钮
  点击后弹出确认对话框,说明导出内容包括:
  照片(base64)/ 位置路径 / 历史记录 / 标签 / 借出状态 / 置顶 / 关联关系。
  确认后选择保存路径,生成 JSON 文件。
  注:API key 等敏感信息不包含在导出文件里。

导入:⌘, → 数据 tab → 从 JSON 导入…
  点击后弹出确认对话框,提示导入会追加到现有数据。
  「导入时跳过重复项目」开关(默认开启):
    · 开启时:名称 + 位置完全相同的物品会被跳过,不重复导入。
    · 关闭时:所有条目强制导入,包含重复物品——
      关闭前请确认,重复物品难以批量删除。

清空所有:⌘, → 数据 tab → 清空所有数据
  危险,需要二次确认(建议导出后再操作)

  ⚙️  偏好设置  (⌘,)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

通用 tab:
  · 语言    跟随系统 / 简体中文 / English
            (SwiftUI 立即生效,部分提示需重启完全生效)
  · 外观    跟随系统 / 浅色 / 深色
  · 菜单栏  显示 / 隐藏菜单栏图标
  · 录入    检测重复 开关
             识别更新语句 开关
             都关 = 录入直接落库不弹任何对话框
  · 自动建议标签  开 / 关(录入后自动挂预设标签)
  · 全局快捷键  默认 ⌥⌘N,可点击按钮进入捕获模式重新绑定
                按任意带 modifier 的组合键完成绑定
                在任何 app 内触发,弹出 QuickEntry 小窗
  · QuickEntry  开 / 关(关掉后全局快捷键同步失效)

标签 tab:
  · 查看所有标签,含已挂物品数
  · 行内改色、改名、删除

AI tab:
  · 用量统计  今天 / 本周 / 本月三列,每列含调用次数 /
              input token / output token / 估算 USD(或 ¥)
              可手动重置;月初自动归零
  · Provider:Claude / 火山引擎(各自独立配置)
  · 模型、API key、Endpoint(可填中转站 URL)
  · 火山引擎 section 另有「输入 ¥/百万 token」
    和「输出 ¥/百万 token」两个可选价格字段(v0.1.7 新增)
  · 自定义 system prompt(可重置为默认)
  · 「测试连接」按钮
  · 「查看图文配置教程(网页)」入口(v0.2.0 新增):跳转网页版保姆级教程

通知 tab:
  · 频率    每天 / 每周 / 每月 1 号
  · 星期    选「每周」后下方出现星期选择器,可选周一到周日
            (默认周一)
  · 时间    时间选择器(默认每天 12:00)
  · 内容模板  自定义通知文案,%@ 替换为物品名

点击通知 banner 会自动在主窗口搜索框填入物品名称并激活窗口,免去手动
查找;App 在前台运行时,通知同样会以 banner 形式弹出。

位置 tab:
  · 树状列出所有 Location,根节点分两个 section:
    🏠 房间(命中房间词词典的根位置,如卧室/客厅/书房/厨房)
    独立位置(其它根位置)
  · 每行可重命名——整棵子树的路径自动跟着变
  · 每行「合并到…」menu:把当前位置 + 子节点 + 物品
    全部迁移到另一个根位置下
  · 删除按钮始终可用(非空节点也能删):
    该位置下的物品自动提升到父节点,数据不丢
  · 顶部多选后出现「批量删除」和「全部合并到…」按钮

数据 tab:
  · 从 JSON 导入
  · 清空所有数据(危险)
